# LabFlow — Market Analysis

## Problem

Educational laboratories often track equipment using spreadsheets, paper sheets, or fragmented messaging between teachers and technicians. This creates several recurring problems:

- equipment gets misplaced or borrowed without traceability
- faults are reported late or informally
- maintenance is reactive instead of planned
- consumables run out unexpectedly
- there is no clean audit trail across users and roles

## Existing alternatives

There are inventory tools, generic asset trackers, and spreadsheet-based workflows, but they usually miss one or more of the following:

- education-focused roles such as professor / technician / student
- QR workflows for fast classroom operations
- local/offline-friendly deployment for schools
- lab-by-lab isolation inside the same app
- integrated borrow + fault + reservation + maintenance flow

## LabFlow advantages

- purpose-built for school and educational laboratories
- local-first SQLite deployment, easy to run on a classroom PC
- companion mobile app for QR flows
- rich operational workflow, not just inventory
- role-based access and lab membership
- modern analytics dashboard and presentation-ready demo data

## Target audience

- school laboratories
- robotics clubs
- STEM coordinators
- teachers and professors
- lab technicians
- school administrators

## Feasibility

The project is feasible both short-term and long-term because:

- it already runs locally in a stable form
- it uses mature open-source technologies
- the architecture supports adding new modules incrementally
